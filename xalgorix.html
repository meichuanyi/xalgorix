Step 1: Install Go 1.24+

If you already have Go 1.24+, skip this step. Otherwise:

# Remove old Go installation
sudo rm -rf /usr/local/go

# Download and install Go 1.24
wget -q https://go.dev/dl/go1.24.2.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.24.2.linux-amd64.tar.gz
rm go1.24.2.linux-amd64.tar.gz

# Add to PATH (add to ~/.bashrc for persistence)
export PATH=/usr/local/go/bin:$HOME/go/bin:$PATH

# Verify
go version  # Should show go1.24.x

Step 2: Install Xalgorix

Option A: Build from Source (Recommended)

git clone https://github.com/xalgord/xalgorix.git
cd xalgorix
make install

Option B: Install via Go

GOPROXY=direct go install -v github.com/xalgord/xalgorix/v4/cmd/xalgorix@latest

Verify Installation

xalgorix --version
💡 Auto-Update: Run xalgorix --update anytime to get the latest version. Xalgorix also auto-updates on every start by downloading binaries from GitHub releases.

The Three Scan Modes

Xalgorix supports three distinct scanning approaches, each designed for different use cases:

🎯 Single Scan — Full 20-phase pentest against a single URL. Perfect for quick, targeted assessments. Enter a URL, click Start.

🔍 DAST Scan — Deep Application Security Testing on specific URLs. Focuses heavily on injection, logic flaws, authentication bypass, and session management.

🌐 Wildcard Scan — The big one. Enumerates all subdomains of a root domain, then queues each subdomain for individual full scanning. Maximum coverage for bug bounty targets.

The 20-Phase Methodology

Every scan follows a structured, comprehensive methodology modeled after professional pentesting frameworks. Here's what happens when you click "Start":

  1. Reconnaissance — Subdomains, ports, directories, technology fingerprinting
  2. Vulnerability Scanning — Nuclei templates, nmap scripts, CVE detection
  3. Content Discovery — Hidden files, backup files, admin panels, sensitive endpoints
  4. SSL/TLS Analysis — Cipher suites, certificates, security headers
  5. Authentication Testing — SQL injection on logins, brute-force, OAuth flaws
  6. Injection Attacks — XSS, SQLi, Command Injection, XXE, SSTI
  7. SSRF Testing — Parameter fuzzing, cloud metadata access, internal service probing
  8. IDOR & Access Control — Horizontal/vertical privilege escalation, BAC
  9. API Security — GraphQL introspection, REST endpoint abuse, rate limiting bypass
  10. File Upload Testing — Extension bypass, webshell upload, MIME type abuse
  11. Remote Code Execution — Deserialization, Log4j, template injection chains
  12. Race Conditions — TOCTOU, business logic concurrency flaws
  13. Subdomain Takeover — Dangling CNAME records, unclaimed cloud services
  14. Email Security — SPF, DKIM, DMARC misconfiguration
  15. Cloud Security — Exposed S3 buckets, Azure blobs, GCP misconfig, K8s
  16. WebSocket Testing — Origin validation, injection, cross-site hijacking
  17. CMS-Specific — WordPress, Joomla, Drupal known vulnerabilities
  18. Broken Link Hijacking — External resource takeover opportunities
  19. Supply Chain — JavaScript library vulnerabilities, dependency confusion
  20. Report Generation — Professional PDF with all findings, CVSS, remediation

Advanced Configuration

Discord Alerts

Get real-time notifications when scans start, vulnerabilities are found, or scans complete. Set up a Discord webhook and add it to your config:

XALGORIX_DISCORD_WEBHOOK=https://discord.com/api/webhooks/YOUR_WEBHOOK_ID/YOUR_TOKEN

Dashboard Authentication

If you're exposing the dashboard on a network (especially a VPS), always enable authentication:

XALGORIX_USERNAME=admin
XALGORIX_PASSWORD=your_strong_password_here

Rate Limiting

Protect your IP from being blocked by targets. Default is 60 requests per 60-second window:

XALGORIX_RATE_LIMIT_REQUESTS=30    # More conservative
XALGORIX_RATE_LIMIT_WINDOW=60       # Window in seconds

Web Search Enhancement

Give the agent web search capabilities for CVE research and tech intelligence:

GEMINI_API_KEY=AIza...
# Falls back to Brave/Google/DuckDuckGo scraping if not set

Running Xalgorix 24/7 (systemd Service)

Want Xalgorix running permanently on a VPS? It has built-in systemd support — no manual service file creation needed:

# Install and start as a systemd service
sudo xalgorix --start

# Check status
sudo systemctl status xalgorix

# Stop / Restart / Uninstall
sudo xalgorix --stop
sudo xalgorix --restart
sudo xalgorix --uninstall
💡 VPS Tip: When running on a remote server, use --port 443 and set up HTTPS with a reverse proxy (nginx/caddy) for secure remote access to the dashboard.

Pro Tips for Maximum Results

1. Use Custom Instructions Wisely

The agent is smart, but context makes it smarter. Always provide relevant instructions:

# For bug bounty programs:
"This is a HackerOne program. Out of scope: DoS, social engineering, rate limiting."

# For authenticated testing:
"Login credentials: admin@example.com / Password123. Test all authenticated endpoints."

# For API-focused testing:
"API Key: Bearer abc123. Focus on GraphQL introspection and IDOR on /api/v2/ endpoints."

# For targeted vulnerability classes:
"Focus exclusively on SQL Injection and SSRF. Skip XSS and info disclosures."

2. Severity Filtering Saves Time

For bug bounty, filter to Critical and High severity only. This prevents the agent from spending time on informational findings that won't earn a bounty. Set this in the Web UI before starting a scan.

3. Wildcard Mode + Discord = Bug Bounty Autopilot

Set up a wildcard scan on a bug bounty target, enable Discord alerts, and let it run overnight. You'll wake up to vulnerability notifications on your phone. This is the closest thing to passive income in bug bounty.

4. Scan Persistence is Your Safety Net

Xalgorix automatically saves scan progress. If your terminal crashes, internet drops, or you need to restart — your scan data is preserved. Resume interrupted scans from the queue management in the Web UI.

5. Multi-Target Queue for Batch Testing

Upload a .txt file with multiple targets (one per line) through the Web UI. Xalgorix queues them and scans sequentially, generating individual reports for each.

Troubleshooting

❌ "xalgorix: command not found"

Make sure $GOPATH/bin is in your PATH. Run: export PATH=$HOME/go/bin:$PATH and add it to your ~/.bashrc. If you built from source, check that make install completed without errors.

❌ "missing required env: XALGORIX_LLM"

Create the file at ~/.xalgorix.env with at minimum XALGORIX_LLM and XALGORIX_API_KEY. Make sure the file is in your home directory, not the project directory.

❌ "API rate limit exceeded"

Lower the rate limiting settings: XALGORIX_RATE_LIMIT_REQUESTS=30. You can also switch to a different LLM provider, or increase XALGORIX_LLM_MAX_RETRIES=10 for more resilience.

❌ Tools failing to install

Ensure you have sudo access. Xalgorix needs root to install packages via apt/snap. If on a restricted system, pre-install tools manually. Run sudo apt update first to refresh package lists.
